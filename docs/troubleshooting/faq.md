# 常见问题

本页汇总了使用 StarHub 时最常见的问题和解决方案。

## 登录相关

### Q: 点击登录后一直跳转失败？

**A:** 检查以下几点：

1. **回调地址匹配**：确保 GitHub OAuth App 的回调地址与当前 URL 完全一致
2. **后端服务运行**：本地开发需确保 `node server/dev-server.js` 正在运行
3. **Client ID 正确**：检查 `src/config/oauth.ts` 中的配置

### Q: 登录成功但看不到仓库？

**A:**

1. 等待同步完成（右上角会显示进度）
2. 检查网络连接
3. 尝试刷新页面

### Q: 如何退出登录？

**A:** 点击右上角头像 → 选择「退出登录」

## 同步相关

### Q: 同步速度很慢？

**A:** 

- GitHub API 每小时限制 5000 次请求
- 首次同步大量仓库需要时间
- 网络状况也会影响速度

### Q: 同步后仓库数量不对？

**A:**

1. 可能还在同步中，等待完成
2. 尝试使用「重新抓取」功能
3. 检查控制台是否有错误

### Q: 如何强制重新同步？

**A:** 点击右上角头像 → 选择「重新抓取」

## 标签相关

### Q: 如何批量添加标签？

**A:**

1. 点击仓库列表顶部的「选择」按钮
2. 勾选要操作的仓库
3. 点击「批量设置分类」
4. 选择标签后确认

### Q: 删除标签后仓库会怎样？

**A:** 仓库不会被删除，只是会失去该标签的关联。

### Q: 一个仓库可以有多个标签吗？

**A:** 是的，StarHub 支持多标签。

## AI 分类相关

### Q: AI 分类需要付费吗？

**A:** 

- AI 服务本身需要费用（按 Token 计费）
- 智谱 AI 提供免费额度
- DeepSeek 和通义千问价格较低

### Q: 分类不准确怎么办？

**A:**

1. 开启「读取 README」选项
2. 减小批次大小
3. 使用更强的模型（如 gpt-4o）
4. 手动调整分类

### Q: API 调用失败？

**A:**

1. 检查 API Key 是否正确
2. 确认账户有余额
3. 检查网络连接
4. 查看控制台错误信息

## 数据相关

### Q: 数据存储在哪里？

**A:** 

- 仓库和标签数据存储在浏览器 IndexedDB
- 设置存储在 localStorage
- 所有数据都在本地，不会上传

### Q: 如何清空所有数据？

**A:**

1. 设置页面 → 数据管理 → 清空所有数据
2. 或在控制台运行：`fetch('/emergency-clear.js').then(r => r.text()).then(eval)`

### Q: 换浏览器/电脑后数据丢失？

**A:**

- StarHub 数据存储在本地浏览器
- 更换浏览器需要重新登录同步
- 目前不支持数据导出/导入

## 性能相关

### Q: 页面加载很慢？

**A:**

1. 首次加载需要下载资源，之后会缓存
2. 清除浏览器缓存后重试
3. 检查网络连接

### Q: 滚动列表卡顿？

**A:**

- StarHub 使用虚拟滚动，理论上不会卡顿
- 尝试使用 Chrome 或 Edge 浏览器
- 检查是否有其他程序占用资源

### Q: 搜索很慢？

**A:**

- 正常情况搜索应该是毫秒级
- 确保数据已完全加载
- 尝试刷新页面

## 其他问题

### Q: 支持手机访问吗？

**A:** StarHub 支持移动端浏览器访问，但体验最佳的是桌面端。

### Q: 支持哪些浏览器？

**A:**

- Chrome 90+
- Firefox 88+
- Edge 90+
- Safari 14+

### Q: 如何反馈 Bug？

**A:** 请在 [GitHub Issues](https://github.com/mengjian-github/starhub/issues) 提交，描述问题和复现步骤。
