# 搜索与筛选

StarHub 提供强大的本地搜索功能，帮你快速找到需要的仓库。

## 基础搜索

### 使用搜索框

在顶部搜索框输入关键词，按 Enter 或直接输入即可搜索。

### 搜索范围

搜索会匹配以下字段：

- 仓库名称
- 仓库描述
- 所有者用户名
- 编程语言

## 搜索语法

### 单关键词

```
react
```

搜索包含 "react" 的仓库。

### 多关键词

```
react hooks
```

搜索同时包含 "react" 和 "hooks" 的仓库。

### 大小写

搜索不区分大小写：

```
React = react = REACT
```

## 标签筛选

### 单标签筛选

点击左侧任意标签，列表只显示该标签下的仓库。

### 组合使用

1. 先点击标签筛选
2. 再使用搜索框缩小范围

示例：先筛选 "Web 开发" 标签，再搜索 "vue"。

## 特殊筛选

### 全部仓库

点击左侧 **全部**，显示所有仓库。

### 未分类仓库

点击左侧 **未分类**，显示没有任何标签的仓库。

这对于：
- 查找需要整理的仓库
- 准备使用 AI 分类

## 搜索技巧

### 使用准确的关键词

| 目标 | 推荐搜索 | 说明 |
|------|----------|------|
| React 相关 | `react` | 直接搜索 |
| Vue 组件库 | `vue component` | 多关键词 |
| Python 工具 | 先筛选 Python 语言 | 结合标签 |

### 按语言搜索

由于搜索会匹配编程语言字段，可以直接搜索：

```
typescript
python
go
rust
```

## 虚拟滚动

StarHub 使用虚拟滚动技术，即使有上千个仓库也能流畅滚动。

### 工作原理

- 只渲染可见区域的仓库卡片
- 滚动时动态加载/卸载
- 内存占用恒定

### 注意事项

- 快速滚动时可能有短暂空白
- 搜索结果即时更新
- 不影响搜索性能

## 排序

当前仓库按以下顺序排列：

1. Star 时间（最新在前）
2. 分类后按标签分组

## 搜索性能

由于数据存储在本地 IndexedDB：

- **搜索延迟**：< 10ms
- **支持数量**：10,000+ 仓库
- **离线可用**：断网也能搜索

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `/` 或 `Ctrl+K` | 聚焦搜索框 |
| `Esc` | 清空搜索并取消聚焦 |
| `Enter` | 确认搜索 |
